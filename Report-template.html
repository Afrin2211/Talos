DQ Check Type Description of Check Severity Definition
Data type mismatches Values in a column do not match the expected data type (e.g., string in an int field). Critical Issue - affects data integrity
Missing or extra columns Columns expected in the dataset are missing, or unexpected columns are present. Critical Issue - breaks expected schema alignment
Nullability violations Fields that should not contain nulls have missing values. Critical Issue - can lead to application crashes or logic failure
Out-of-range values Values fall outside the acceptable minimum or maximum defined for the field. Medium Issue - leads to inaccurate insights or statistical distortions
Format errors Values do not conform to the required format (e.g., date format, phone number). Medium Issue - impacts standard parsing and validation
Uniqueness violations Duplicate values are found in columns expected to have unique entries. Critical Issue - risks data integrity and incorrect joins
Reference integrity issues Foreign key values do not match any entry in the referenced table. Critical Issue - causes broken relationships in relational datasets
Cross-field inconsistencies Values in one field conflict logically with values in another (e.g., end date before start date). Medium Issue - causes logical conflicts between related fields
Chronological errors Temporal data is not in a logical sequence (e.g., birth date after registration). Medium Issue - can mislead time-series analysis
Standardization problems Data does not follow consistent naming or formatting conventions (e.g., inconsistent case or abbreviations). Low Issue - reduces consistency in categorical or textual data
